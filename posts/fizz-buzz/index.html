The correct way to write fizz-buzz

July 31, 2017
javascript youtube code

Youtube boredom

Often during downtime, I find myself wondering to youtube to pass the time. Earlier today that lead me to Tom Scott’s new video on the usage of fizz-buzz as a programmer interview question:

During the video, Tom discussed about how programmers write their solution in a multitude of different ways. I thought it would be interesting to try my own hand at it, and see what I came up with…

var values = {
    fizz: 3,
    buzz: 5
};
var output = "";
for (var i = 1; i<=100; i++) {
    output = "";
    for(var key in values) {
        if(i % values[key] == 0) {
            output += key;
        }
    }
    if (output == "") {
        output = i;
    }
    console.log(output);
}

Essentially, It showed me that the years working at 7Cav.us as a developer in a PHP heavy stack, really impacted me on with regards to storing and accessing data. Even with small command line tools like this, I prefer to keep data separate from the logic and dip my toe in as and when I need it.

Even though it makes the solution longer than that of some wiz kid dev, I feel this makes it incredibly easier to add new values to compare against, rather than a long list of if’s or a switch instead.

Code efficiency is one thing. Code extendability is key.
